# GSD Orchestrator 보안 가이드

GSD Orchestrator는 `claude -p --dangerously-skip-permissions`로 AI에게 셸 명령 실행 권한을 부여합니다. 이 문서는 로컬 환경과 서버 환경에서의 보안 설정을 안내합니다.

---

## 위험 요소

| 요소 | 설명 |
|------|------|
| 셸 명령 실행 | Claude Code가 파일 생성/수정/삭제, 패키지 설치 등을 직접 수행 |
| 네트워크 접근 | Claude Code가 외부 API 호출, 패키지 다운로드 가능 |
| 파일 시스템 접근 | `working_dir` 외부의 파일에도 접근 가능 |
| 봇 토큰 노출 | `.env`에 저장된 토큰이 유출될 경우 봇 탈취 가능 |

---

## 레벨 1: 최소 보호 (권장)

별도 사용자 계정으로 실행하여 시스템 파일과 개인 파일을 격리합니다.

```bash
# macOS: 별도 사용자 생성
sudo dscl . -create /Users/gsdbot
sudo dscl . -create /Users/gsdbot UserShell /bin/zsh
sudo dscl . -create /Users/gsdbot NFSHomeDirectory /Users/gsdbot
sudo dscl . -create /Users/gsdbot UniqueID 550
sudo dscl . -create /Users/gsdbot PrimaryGroupID 20
sudo createhomedir -c -u gsdbot

# Linux: 별도 사용자 생성
sudo useradd -m -s /bin/bash gsdbot

# 프로젝트를 gsdbot 사용자로 실행
sudo -u gsdbot bash -c 'cd /home/gsdbot/GSD-Orchestrator && ./start.sh'
```

**효과:**
- gsdbot 사용자 권한으로만 파일 접근 가능
- 관리자/개인 파일에 접근 불가
- `sudo` 권한 없음

## 레벨 2: Docker 격리 (강력)

컨테이너 내에서 실행하여 호스트 시스템과 완전히 격리합니다.

### Dockerfile

```dockerfile
FROM python:3.12-slim

# 시스템 의존성
RUN apt-get update && apt-get install -y --no-install-recommends \
    git nodejs npm \
    && rm -rf /var/lib/apt/lists/*

# Claude Code CLI 설치
RUN npm install -g @anthropic-ai/claude-code

# 비root 사용자 생성
RUN useradd -m -s /bin/bash gsdbot
USER gsdbot
WORKDIR /home/gsdbot/app

# 프로젝트 복사 및 설치
COPY --chown=gsdbot:gsdbot . .
RUN python -m venv .venv && \
    .venv/bin/pip install -e .

# 디렉토리 생성
RUN mkdir -p messages/{inbox,outbox,sent,error,archive} logs workspace

# 진입점
CMD [".venv/bin/python", "-m", "gsd_orchestrator"]
```

### docker-compose.yml

```yaml
version: "3.8"

services:
  gsd-orchestrator:
    build: .
    container_name: gsd-orchestrator
    restart: unless-stopped
    env_file: .env
    volumes:
      # 작업 디렉토리만 마운트 (호스트 파일 접근 차단)
      - ./workspace:/home/gsdbot/app/workspace
      # 메시지/로그 영속화
      - ./messages:/home/gsdbot/app/messages
      - ./logs:/home/gsdbot/app/logs
    # 불필요한 권한 제거
    security_opt:
      - no-new-privileges:true
    # 읽기 전용 루트 파일시스템 (workspace/messages/logs/tmp 제외)
    read_only: true
    tmpfs:
      - /tmp
```

```bash
# 실행
docker compose up -d

# 로그 확인
docker compose logs -f

# 중지
docker compose down
```

**효과:**
- 호스트 파일 시스템에 `workspace/`, `messages/`, `logs/`만 접근 가능
- 컨테이너 내부에서 호스트 시스템 명령 실행 불가
- `no-new-privileges`로 권한 상승 차단
- `read_only`로 시스템 파일 변조 방지

## 레벨 3: 네트워크 제한 (최고)

Docker + 네트워크 정책으로 외부 접근을 최소화합니다.

```yaml
# docker-compose.yml에 추가
services:
  gsd-orchestrator:
    # ... 위의 설정에 추가
    networks:
      - gsd-net

networks:
  gsd-net:
    driver: bridge
    # 필요 시 외부 접근을 특정 도메인으로 제한
    # (iptables/nftables 규칙으로 세밀하게 제어 가능)
```

허용이 필요한 외부 도메인:
- `api.anthropic.com` — Claude API
- `api.telegram.org` — Telegram 봇
- `wss-primary.slack.com` — Slack Socket Mode
- `registry.npmjs.org` — npm 패키지 (GSD 설치 시)

## .env 보안

```bash
# 권한 제한 (소유자만 읽기)
chmod 600 .env

# .gitignore에 포함 확인
grep -q '.env' .gitignore || echo '.env' >> .gitignore
```

`.env`에 저장되는 민감 정보:
- `TELEGRAM_BOT_TOKEN` — 유출 시 봇 탈취 가능
- `TELEGRAM_CHAT_ID` — 유출 시 메시지 가로채기 가능
- `SLACK_BOT_TOKEN` / `SLACK_APP_TOKEN` — 유출 시 워크스페이스 접근 가능

## working_dir 격리

`config.yaml`의 `claude.working_dir`은 Claude Code가 작업하는 디렉토리입니다. 이 디렉토리를 프로젝트와 분리하여 Claude가 오케스트레이터 코드를 수정하지 못하도록 합니다.

```yaml
claude:
  working_dir: workspace    # 기본값: 프로젝트 하위 디렉토리
```

프로덕션 환경에서는 별도 경로를 지정합니다:

```yaml
claude:
  working_dir: /home/gsdbot/workspace    # 오케스트레이터와 분리
```

## 보안 체크리스트

| 항목 | 확인 |
|------|------|
| `.env` 파일 권한이 `600`인가? | `chmod 600 .env` |
| `.env`가 `.gitignore`에 포함되어 있는가? | `grep .env .gitignore` |
| 관리자가 아닌 별도 사용자로 실행하는가? | `whoami` |
| `working_dir`이 시스템 디렉토리가 아닌가? | config.yaml 확인 |
| 인스턴스당 봇 토큰이 고유한가? | 토큰 중복 사용 금지 |
| Docker 사용 시 볼륨 마운트가 최소한인가? | docker-compose.yml 확인 |
