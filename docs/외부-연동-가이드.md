# GSD Orchestrator 외부 연동 가이드

gsd_orchestrator는 단독 실행뿐 아니라 외부 프로젝트에 패키지로 설치하여 연동할 수 있습니다.
호스트 앱은 인터페이스 2개로 메시징 + AI 기능을 활용합니다.

---

## 설치

git 저장소에서 설치:

```bash
pip install git+https://github.com/sevengoe/GSD-Orchestrator.git
```

호스트 앱 pyproject.toml:

```toml
dependencies = [
    "gsd-orchestrator @ git+https://github.com/sevengoe/GSD-Orchestrator.git",
]
```

Slack 사용 시:

```bash
pip install "gsd-orchestrator[slack] @ git+https://github.com/sevengoe/GSD-Orchestrator.git"
```

---

## 연동 모드

| 모드 | 방식 | 기동 | 사용 시점 |
|------|------|------|-----------|
| **Library** | Python API (`send()`, `on_result()`) | 호스트 앱 코드에서 `run()` 또는 `start()` 호출 | 호스트 앱에 패키지로 import하여 같은 프로세스에서 실행 |
| **독립 프로세스** | 파일 기반 (`inbox/`, `sent/`) | `./start.sh`로 별도 프로세스 기동 | GSD-Orchestrator가 별도 프로세스로 실행 중일 때 |

두 모드 모두 GSD-Orchestrator의 기동이 필요합니다. Library 모드는 호스트 앱 프로세스 안에서 함께 실행되고, 독립 프로세스 모드는 별도 프로세스로 먼저 기동해야 합니다.

---

## 독립 프로세스 연동 (파일 기반)

GSD-Orchestrator가 독립 프로세스로 실행 중일 때, 외부 앱은 파일 시스템을 통해 연동합니다.
별도 패키지 설치 없이 JSON 파일 읽기/쓰기만으로 동작합니다.

### 요청 보내기 — inbox/에 JSON 작성

외부 앱이 `messages/inbox/`에 JSON 파일을 작성하면 GSD-Orchestrator가 자동으로 처리합니다.

```python
import json
import uuid
from datetime import datetime, timezone, timedelta
from pathlib import Path

KST = timezone(timedelta(hours=9))
INBOX_DIR = Path("/path/to/GSD-Orchestrator/messages/inbox")

def send_request(text: str, mode: str = "default") -> Path:
    now = datetime.now(KST)
    short_id = uuid.uuid4().hex[:8]
    filename = f"{now.strftime('%Y%m%d_%H%M%S')}_{short_id}.json"

    data = {
        "id": str(uuid.uuid4()),
        "source": {
            "channel_type": "external",
            "channel_id": "my-app",
            "user_id": "my-app",
            "user_name": "My App",
            "message_id": 0,
            "thread_ts": None,
        },
        "chat_id": "my-app",
        "message_id": 0,
        "keyword": text[:20],
        "mode": mode,
        "request": {
            "text": text,
            "timestamp": now.isoformat(),
        },
        "response": None,
    }

    # 원자적 쓰기: .tmp → rename
    tmp = INBOX_DIR / f".{filename}.tmp"
    tmp.write_text(json.dumps(data, ensure_ascii=False, indent=2))
    target = INBOX_DIR / filename
    tmp.rename(target)
    return target
```

**주의사항:**
- 반드시 `.tmp` → `rename` 패턴으로 원자적 쓰기 (반쯤 쓰인 파일 방지)
- `mode`는 `"default"` (자동 분류) 또는 `"gsd"` (GSD 강제 라우팅)

### 결과 받기 — sent/ 모니터링

처리 완료된 메시지는 `messages/sent/`에 저장됩니다. `response` 필드가 채워져 있습니다.

```python
import json
import time
from pathlib import Path

SENT_DIR = Path("/path/to/GSD-Orchestrator/messages/sent")

def poll_results(interval: int = 5):
    seen = set()
    while True:
        for f in SENT_DIR.glob("*.json"):
            if f.name in seen:
                continue
            seen.add(f.name)
            data = json.loads(f.read_text())
            if data.get("source", {}).get("channel_type") == "external":
                print(f"결과: {data['response']}")
        time.sleep(interval)
```

**참고:** `sent/`의 파일은 주기적으로 `archive/`로 이동됩니다. 실시간 처리가 필요하면 폴링 간격을 짧게 설정하세요.

### 메시지 라이프사이클

```
외부 앱 → inbox/ → .processing → Claude/GSD 처리 → outbox/ → 채널 발송 → sent/
```

### 에러 처리

처리 실패 시 `messages/error/`로 이동됩니다. 재처리:
```bash
mv messages/error/파일.json messages/inbox/
```

## Library 연동 (같은 프로세스)

### 1. ChannelSender — 메시지 전송 요청

**방향**: 호스트 앱 → gsd_orchestrator

호스트 앱이 활성 채널(Telegram/Slack)에 메시지를 보냅니다.

```python
sender = orchestrator.channel_sender

# 모든 채널에 즉시 발송
await sender.send("[Agent-A] 작업 완료: 결과 요약")

# 특정 채널에 즉시 발송
await sender.send("긴급 알림", channel_type="telegram")

# outbox 대기열에 저장 (재시도 보장)
sender.enqueue("백그라운드 알림")

# 특정 채널 대기열
sender.enqueue("Slack 전용 알림", channel_type="slack")
```

| 메서드 | 특징 |
|---|---|
| `send(text, channel_type?)` | 즉시 발송. 네트워크 실패 시 False 반환 |
| `enqueue(text, channel_type?)` | outbox 파일 큐. 최대 3회 재시도, 유실 방지 |

`channel_type` 미지정 시 모든 활성 채널에 발송.

### 2. on_result — 메시지 처리 결과

**방향**: gsd_orchestrator → 호스트 앱

Claude/GSD가 처리 완료(또는 실패)하면 콜백으로 결과를 전달합니다.

```python
async def my_handler(source: dict, request_text: str,
                     response_text: str, status: str):
    if status == "success":
        await db.save(source, request_text, response_text)
    elif status == "error":
        log.error(f"실패: {response_text}")
    elif status == "blocked":
        log.info(f"GSD 블로킹: {response_text}")

orchestrator.on_result(my_handler)
```

**콜백 인자:**

| 인자 | 내용 |
|---|---|
| `source` | 요청자 정보 (`channel_type`, `channel_id`, `user_id`, `user_name`, ...) |
| `request_text` | 사용자 원본 요청 |
| `response_text` | 처리 결과 (실패 시 에러 메시지, 블로킹 시 질문 텍스트) |
| `status` | `"success"` \| `"error"` \| `"blocked"` |

콜백 미등록 시 채널 발송만 수행 (단독 실행과 동일).

### 사용법

#### config.yaml 작성

호스트 앱 디렉토리에 config.yaml을 작성합니다:

```yaml
channels:
  telegram:
    enabled: true
    chat_id: ""          # .env TELEGRAM_CHAT_ID 사용

broadcast:
  snippet_length: 500

polling:
  inbox_check_interval: 10
  outbox_interval: 3
  progress_interval: 30

archive:
  message_retention_days: 30

claude:
  timeout: 300
  cooldown_retry_minutes: 10
  max_session_turns: 20
  working_dir: /path/to/your/project   # Claude 작업 디렉토리

gsd:
  enabled: true
  auto_classify: true
  classify_model: haiku

paths:
  inbox_dir: messages/inbox
  outbox_dir: messages/outbox
  sent_dir: messages/sent
  error_dir: messages/error
  archive_dir: messages/archive
```

#### 블로킹 방식 (메인 루프 위임)

```python
from gsd_orchestrator.orchestrator import Orchestrator
from gsd_orchestrator.config import Config

config = Config.load("config.yaml")
orchestrator = Orchestrator(config)

# 인터페이스 연결
orchestrator.on_result(my_handler)
sender = orchestrator.channel_sender

# gsd_orchestrator가 메인 루프 점유
orchestrator.run()
```

#### 비동기 방식 (기존 이벤트 루프에 추가)

호스트 앱이 자체 이벤트 루프(FastAPI 등)를 운영 중인 경우:

```python
from gsd_orchestrator.orchestrator import Orchestrator
from gsd_orchestrator.config import Config

config = Config.load("config.yaml")
orchestrator = Orchestrator(config)
orchestrator.on_result(my_handler)

# 호스트 앱의 이벤트 루프에 백그라운드로 추가
await orchestrator.start()

sender = orchestrator.channel_sender
# ... 호스트 앱 로직 계속 ...

# 종료 시
await orchestrator.stop()
```

### 호출 흐름

```
채널 메시지 수신
  ↓
gsd_orchestrator 기본 동작
  ├── /gsd, /reset, /status, /resume 커맨드 처리
  ├── inbox 저장 + 전채널 "작업중입니다" 브로드캐스트
  ├── inbox_processor → Claude/GSD 실행
  └── outbox 조립 → 채널 발송
  ↓
on_result 콜백 (등록된 경우)
  → source, request_text, response_text, status 전달
```

```
호스트 앱 백그라운드 프로세스
  ↓
sender.send("즉시 발송")       → 활성 채널로 발송
sender.enqueue("대기열 발송")  → outbox 경유, 재시도 보장
```

---

## 주의사항

- gsd_orchestrator는 호스트 앱을 모릅니다 (의존성 없음)
- `.env` 파일에 채널 토큰이 필요합니다 (TELEGRAM_BOT_TOKEN 등)
- `claude` CLI가 PATH에 있어야 합니다
- `paths`의 디렉토리는 자동 생성됩니다
- `on_result` 콜백에서 예외가 발생해도 gsd_orchestrator 동작에 영향 없습니다
